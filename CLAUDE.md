## 출력 스키마 (docs/data/feed.json)
```json
{
  "last_updated": "2026-06-24T07:00:00Z",
  "items": [
    {
      "id": "sha256-앞8자리",
      "date": "2026-06-24",
      "category": "🚀 릴리즈",
      "title_ko": "한국어 제목",
      "summary_ko": "리드 — 2~3문장 한입 요약. 사실만, 과장 없이.",
      "body_ko": "기사 본문 — 원문을 충실히 풀어쓴 기사형. 문단은 \\n\\n 으로 구분.",
      "source": "Anthropic Blog",
      "url": "https://...",
      "importance": "high",
      "tags": ["Claude", "API"]
    }
  ]
}
```

## 규칙
- 각 카드는 **리드 + 기사 본문** 2단 구성 (사이트에서 리드가 먼저 보이고 '더보기'로 본문이 펼쳐짐)
  - `summary_ko` (리드): 사실 중심 2~3문장 한입 요약. 카드 첫 화면에 노출되는 부분.
  - `body_ko` (본문): 원문을 실제로 읽고 충실히 풀어쓴 기사형 본문. 빈 줄(`\n\n`)로 문단 구분, 보통 2~4문단.
    - 핵심 사실·수치·인물·맥락을 담되 군더더기 없이. 본문만 읽어도 원문 내용을 파악할 수 있게.
    - **원문을 읽지 않고 본문을 지어내지 말 것.** 원문 접근이 안 되면 비워두고 사람에게 표시한다.
- 톤은 사실 중심, 과장·감탄사 없이 (출처가 한 표현을 인용한 경우는 보도 주체를 밝히면 허용)
- **`url`은 개별 기사를 직접 가리켜야 한다 — 회사 대문/랜딩 페이지(/news, /blog 등) 금지**
- 하루 최종 카드 수: 5~15개 (15개 초과 금지 — 품질 우선)
- feed.json은 최신 30일치만 유지 (오래된 항목은 archive로 이동)
- 항상 한국어로 Joy와 대화
- **게시(push) 직전 `curation-quality-gate` 스킬로 품질 점검 후 올린다** (문체·과장·중복·출처·스키마·대문URL 검사)

## 30일 롤오버 절차
게시(push) 직전, feed.json의 `items`에서 `date`가 오늘 기준 30일을 초과한 항목을 정리한다.
1. 30일 초과 항목을 골라 **발행 월별**로 `docs/data/archive/YYYY-MM.json`에 합친다
   - 해당 월 파일이 이미 있으면 items 배열에 append, 없으면 새로 생성 (스키마는 feed.json items와 동일)
2. feed.json `items`에서 해당 항목들을 제거 (최신순 유지)
3. archive는 보관 전용 — 사이트(app.js)는 읽지 않으므로 형식만 맞으면 됨
4. 모든 JSON은 커밋 전 유효성 확인 (`python3 -m json.tool`)

> 현재 카드는 전부 2026-06월 발행이라 아직 롤오버 대상 없음. archive 폴더만 준비된 상태.
